## Naming Conventions

### Files
- **Skill files**: lowercase with underscores/hyphens
- **Sample outputs**: `KYC_Decision_{CustomerName}.md`
- **Documentation**: UPPERCASE for planning docs

### State Names
- **PascalCase**: `InitialInformationGathering`, `EntityUnwrapping`
- **Descriptive**: Name indicates purpose
- **No abbreviations**: Except for well-known terms (KYC, EDD, PEP)

### Variables
- **snake_case**: `customer_basic_info`, `sanctions_results`
- **Descriptive**: Purpose clear from name
- **Prefixed by category**: `individual_kyc_data`, `entity_structure_list`
